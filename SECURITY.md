## Сообщение об уязвимости

Если вы обнаружили уязвимость безопасности, **пожалуйста, не создавайте публичный Issue**. Вместо этого сообщите об уязвимости ответственно.

### Как сообщить

**Email:** bivlked@example.com  
**Тема:** `[SECURITY] Краткое описание уязвимости`

### Что включить в сообщение

- **Описание уязвимости** - подробное описание проблемы
- **Шаги для воспроизведения** - как воспроизвести уязвимость
- **Потенциальное влияние** - какое влияние может иметь уязвимость
- **Предложения по исправлению** - если у вас есть идеи по исправлению (опционально)
- **Версия проекта** - какая версия затронута

### Пример сообщения

```
Тема: [SECURITY] Уязвимость в обработке XML

Описание:
Обнаружена уязвимость в функции ParseXML, которая может привести к...

Шаги для воспроизведения:
1. ...
2. ...

Потенциальное влияние:
- ...

Версия: 1.0.0
```

## Процесс обработки

### 1. Подтверждение (в течение 48 часов)

- Мы подтвердим получение вашего сообщения
- Сообщим о начале оценки уязвимости

### 2. Оценка (в течение 7 дней)

- Оценим серьезность уязвимости
- Определим затронутые версии
- Разработаем план исправления

### 3. Исправление (в зависимости от серьезности)

**Критические уязвимости:**
- Исправление в течение 7 дней
- Немедленный релиз патча

**Высокие уязвимости:**
- Исправление в течение 14 дней
- Релиз в следующем патч-релизе

**Средние/Низкие уязвимости:**
- Исправление в следующем релизе
- Обычный цикл релиза

### 4. Disclosure (раскрытие информации)

После исправления уязвимости:

- Опубликуем информацию об уязвимости в CHANGELOG.md
- Создадим Security Advisory на GitHub (если применимо)
- Поблагодарим контрибьютора за ответственное сообщение
- Укажем контрибьютора в списке благодарностей (если он согласен)

## Классификация уязвимостей

### Критические

- Удаленное выполнение кода (RCE)
- Утечка конфиденциальных данных
- Полный обход аутентификации

### Высокие

- Локальное выполнение кода
- Частичный обход аутентификации
- Значительная утечка данных

### Средние

- Отказ в обслуживании (DoS)
- Незначительная утечка информации
- Проблемы с валидацией входных данных

### Низкие

- Информационные утечки
- Проблемы с конфигурацией
- Незначительные проблемы безопасности

## Дополнительная информация

### Что НЕ является уязвимостью

- Проблемы с конфигурацией на стороне пользователя
- Проблемы с зависимостями, которые уже исправлены в последних версиях
- Проблемы, требующие физического доступа к устройству
- Социальная инженерия

### Безопасность зависимостей

Мы регулярно проверяем зависимости на наличие известных уязвимостей:

- Используем `go mod` для управления зависимостями
- Следим за обновлениями безопасности
- Используем Dependabot для автоматических обновлений
