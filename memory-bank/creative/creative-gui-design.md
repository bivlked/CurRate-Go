## 1️⃣ PROBLEM

### Описание проблемы
Необходимо спроектировать GUI для desktop приложения CurRate-Go на Wails v2, которое:
- Сохраняет функциональность оригинального Python приложения (Tkinter)
- Улучшает пользовательский опыт (компактнее, функциональнее, красивее, удобнее)
- Работает на Windows 11+ с минимальным размером дистрибутива
- Обеспечивает интуитивный и современный интерфейс

### Требования

#### Функциональные требования:
- ✅ Заголовок окна: "[💱] Конвертер валют (c) BiV 2025 г."
- ✅ Поле ввода даты (формат ДД.ММ.ГГГГ)
- ✅ Кнопка календаря с визуальным выбором даты
- ✅ Выделение выходных дней красным цветом в календаре
- ✅ Радиокнопки выбора валюты (USD/EUR)
- ✅ Поле ввода суммы (числовое)
- ✅ **Live preview курса:** автоматический показ курса выбранной валюты при изменении даты
- ✅ Кнопка "Конвертировать"
- ✅ Поле отображения результата конвертации
- ✅ Кнопка "Копировать в буфер обмена"
- ✅ **Строка состояния:** отображение ошибок и статуса (без всплывающих окон)
- ✅ Обработка ошибок с информативными сообщениями на русском языке

#### Технические требования:
- Размер окна: 800×650 пикселей (компактное)
- Платформа: Windows 11+ (WebView2)
- Технологии: HTML/CSS/Vanilla JavaScript (без фреймворков)
- Производительность: мгновенная реакция на действия пользователя
- Доступность: поддержка клавиатурной навигации

#### UX требования:
- Интуитивность: пользователь должен понять интерфейс без инструкций
- Компактность: все элементы на одном экране
- Визуальная иерархия: важные элементы выделены
- Обратная связь: визуальная реакция на действия
- Современный дизайн: соответствует стилю Windows 11
- **Удобство:** интерфейс должен быть максимально удобным для использования
- **Красота:** современный и привлекательный внешний вид
- **Функциональность:** все необходимые функции доступны и работают эффективно
- **Live preview:** возможность просматривать курсы, просто выбирая даты

### Ограничения
- **Технологические:** Wails v2, WebView2, Vanilla JS (без React/Vue)
- **Платформенные:** Только Windows 11+ (не кроссплатформенное)
- **Размер:** Минимальный размер дистрибутива (без тяжелых библиотек)
- **Совместимость:** Сохранение функциональности оригинального Python приложения

### Детальный анализ

#### Option A: Классический вертикальный layout

**Плюсы:**
- ✅ Максимальное соответствие оригинальному Python приложению
- ✅ Простая реализация (прямолинейная структура)
- ✅ Привычный интерфейс для пользователей оригинальной версии
- ✅ Легко читается и понимается
- ✅ Низкая сложность разработки

**Минусы:**
- ❌ Устаревший визуальный стиль
- ❌ Менее компактный (больше вертикального пространства)
- ❌ Менее современный вид
- ❌ Не использует преимущества WebView2 (современный CSS)

**Сложность реализации:** Низкая (2-3 дня)  
**Риски:** Минимальные

#### Option B: Двухколоночный layout

**Плюсы:**
- ✅ Хорошее разделение ввода и результата
- ✅ Компактное использование пространства
- ✅ Логичная группировка элементов
- ✅ Улучшенная читаемость результата

**Минусы:**
- ❌ Отличается от оригинального дизайна
- ❌ Может быть неудобен на узких экранах
- ❌ Требует адаптивной верстки
- ❌ Средняя сложность реализации

**Сложность реализации:** Средняя (3-4 дня)  
**Риски:** Средние (адаптивность)

#### Option C: Современный карточный layout

**Плюсы:**
- ✅ Современный и красивый дизайн в стиле Windows 11
- ✅ Отличная визуальная иерархия
- ✅ Четкое разделение функциональных блоков
- ✅ Высокая читаемость результата
- ✅ Профессиональный внешний вид
- ✅ Хорошая масштабируемость для будущих функций

**Минусы:**
- ❌ Отличается от оригинального дизайна
- ❌ Более высокая сложность реализации (CSS карточки, тени, градиенты)
- ❌ Больше кода для стилизации
- ❌ Требует больше времени на разработку

**Сложность реализации:** Средне-высокая (4-5 дней)  
**Риски:** Средние (время разработки)

#### Option D: Минималистичный однострочный layout

**Плюсы:**
- ✅ Максимальная компактность
- ✅ Быстрый ввод данных
- ✅ Минимальное использование пространства
- ✅ Современный минималистичный стиль

**Минусы:**
- ❌ Сильно отличается от оригинального дизайна
- ❌ Менее удобен для новичков
- ❌ Плохая читаемость результата (все в одну строку)
- ❌ Сложность размещения календаря
- ❌ Не подходит для длинных результатов

**Сложность реализации:** Средняя (3-4 дня)  
**Риски:** Высокие (UX проблемы)

### Ключевые инсайты

1. **Соответствие оригиналу vs Современность:**
   - Option A максимально соответствует оригиналу, но выглядит устаревшим
   - Option C предлагает лучший баланс современности и функциональности

2. **UX и читаемость:**
   - Option C и B обеспечивают лучшую читаемость результата
   - Option D проблематичен для длинных результатов

3. **Сложность реализации:**
   - Option A самая простая, но не использует возможности современного веба
   - Option C требует больше работы, но дает лучший результат

4. **Масштабируемость:**
   - Option C лучше всего подходит для будущих улучшений
   - Option D ограничен в возможностях расширения

## 4️⃣ DECISION

### Выбранный вариант: **Option C - Современный карточный layout с улучшениями**

### Обоснование решения

1. **Баланс между оригиналом и современностью:**
   - Сохраняет все элементы оригинального интерфейса
   - Улучшает визуальное представление с помощью карточек
   - Соответствует современным стандартам Windows 11

2. **Преимущества UX:**
   - Четкая визуальная иерархия через карточки
   - Улучшенная читаемость результата (отдельная карточка)
   - Профессиональный внешний вид
   - Интуитивная группировка элементов

3. **Технические преимущества:**
   - Использует возможности современного CSS (Flexbox, Grid, shadows)
   - Хорошая масштабируемость для будущих функций
   - Легко адаптируется под разные размеры экрана
   - Поддерживает темную тему Windows 11 (будущее расширение)

4. **Компромиссы:**
   - Небольшое увеличение времени разработки (оправдано качеством)
   - Больше CSS кода (но структурированного и поддерживаемого)

### Модификации Option C

Для лучшего соответствия требованиям, Option C будет модифицирован:

1. **Упрощение карточек:**
   - Легкие тени вместо тяжелых градиентов
   - Минималистичные границы
   - Чистый белый/светлый фон

2. **Оптимизация компактности:**
   - Уменьшение отступов между карточками
   - Компактные размеры элементов
   - Эффективное использование пространства

3. **Сохранение функциональности:**
   - Все элементы оригинального интерфейса сохранены
   - Порядок элементов логичен и интуитивен
   - Календарь с выделением выходных красным

```javascript
// live-preview.js
let previewTimeout;

function setupLivePreview() {
    const dateInput = document.getElementById('date-input');
    const calendar = document.getElementById('calendar');
    
    // Debounce для оптимизации
    dateInput.addEventListener('input', debounce(updateRatePreview, 300));
    calendar.addEventListener('datechange', debounce(updateRatePreview, 300));
}

async function updateRatePreview() {
    const date = getSelectedDate();
    const currency = getSelectedCurrency();
    
    if (!date || !currency) return;
    
    showLoadingIndicator();
    
    try {
        const rate = await window.go.main.App.GetRate(currency, date);
        displayRatePreview(rate);
    } catch (error) {
        showErrorInStatusBar(error.message);
    } finally {
        hideLoadingIndicator();
    }
}

function debounce(func, wait) {
    let timeout;
    return function executedFunction(...args) {
        const later = () => {
            clearTimeout(timeout);
            func(...args);
        };
        clearTimeout(timeout);
        timeout = setTimeout(later, wait);
    };
}
```

```javascript
// status-bar.js
function showStatus(message, type = 'info', duration = 3000) {
    const statusBar = document.getElementById('status-bar');
    const statusMessage = document.getElementById('status-message');
    
    statusMessage.className = `status-message ${type}`;
    statusMessage.querySelector('.status-text').textContent = message;
    
    // Автоматическое скрытие для success и info
    if (type === 'success' || type === 'info') {
        setTimeout(() => {
            if (statusMessage.classList.contains(type)) {
                showStatus('Готов к работе', 'success', 0);
            }
        }, duration);
    }
}

function showError(message) {
    showStatus(message, 'error', 0); // Ошибки не скрываются автоматически
}

function showSuccess(message) {
    showStatus(message, 'success', 3000);
}

function showWarning(message) {
    showStatus(message, 'warning', 5000);
```
